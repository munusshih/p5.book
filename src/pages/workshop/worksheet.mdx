This workshop teaches you how to make a printable book with code. We'll use p5.js for drawing and p5.book for paging, previewing, and exporting PDF.

p5.book adds one key idea on top of p5.js: **`draw()` is treated as one book page per capture.**

## <kbd>A</kbd> — primer

If you don't know what [p5.js](https://p5js.org/) is: it's a JavaScript library that makes coding visual art easier. It provides a simple API for drawing shapes, handling user input, and creating animations.

You write your code in two main functions: `setup()` runs once to set up your canvas and variables, while `draw()` runs in a loop to create dynamic visuals. p5.js is great for learning programming concepts and making interactive graphics.

```js
function setup() {
  createCanvas(400, 300); // one-time setup: create a canvas to draw on
}

function draw() {
  background(random(255), random(255), random(255));
  fill(255, 0, 0);
  circle(width / 2, height / 2, 50);
  // draw() runs in a loop, so this circle will be drawn every frame
}
```

Hopefully you have some experience with p5.js already, but if not, no worries! We will cover the basics as we go, and you can also check out the [p5.js reference](https://p5js.org/reference/) for more details on how to use it.

I also recommend opening the p5.js editor in another tab so you can follow along with the code examples and experiment on your own. You can find it at [editor.p5js.org](https://editor.p5js.org/).

## <kbd>C</kbd> — which page am I on?

Use page state to control progression:

| function          | what it does                                     |
| ----------------- | ------------------------------------------------ |
| `book.pageNumber` | current page, 1-based                            |
| `book.page`       | current page, 0-based (good for indexing arrays) |
| `book.totalPages` | total page count                                 |
| `book.progress`   | 0 on first page → 1 on last page                 |

<Diagram

  src="/diagrams/worksheet/c-page-state.svg"

  caption="Use pageNumber/page/progress to create visual change over time."

```js
function draw() {
  background("orange");

  fill(0);
  textFont("Georgia");
  textStyle(BOLD);
  textSize(width / 2);
  textAlign(CENTER, CENTER);
  text(book.pageNumber, width / 2, height / 2);

  book.addPage();
}
```

## <kbd>D</kbd> — cover and back cover

Treat the first and last page differently:

```js
if (book.isFirstPage()) {
  // cover
} else if (book.isLastPage()) {
  // back cover
} else {
  // inside pages
}
```

<Diagram
  label="Cover Structure"
  src="/diagrams/worksheet/d-cover-logic.svg"
  alt="Page sequence with cover, interior pages, and back cover"
  caption="Page 1 is cover, middle pages are interior, and the last page is back cover."
/>

```js
function draw() {
  if (book.isFirstPage()) {
    background("orange");
    fill(255);
    text("COVER", width / 2, height / 2);
  } else if (book.isLastPage()) {
    background("black");
    fill(255);
    text("THE END", width / 2, height / 2);
  } else {
    background(255);
    fill(0);
    text(book.pageNumber, width / 2, height / 2);
  }

  textAlign(CENTER, CENTER);
  textSize(60);
  book.addPage();
}
```

## <kbd>E</kbd> — spreads

A spread is a facing left-right page pair. With spread mode, interior pages are paired while cover/back remain solo pages.

```js
function setup() {
  book = createBook(4, 6, 12);
  book.setSpread(true);
}
```

<Diagram
  label="Spread Pairing"
  src="/diagrams/worksheet/e-spreads.svg"
  alt="Spread layout showing interior left/right pairing"
  caption="Spread mode pairs interior pages as facing left/right pages."
/>

Use page-side helpers:

```js
if (book.isLeftPage()) {
  background("blue");
} else if (book.isRightPage()) {
  background("pink");
}
```

Optional reading direction:

```js
book.setDirection("rtl");
```

## <kbd>F</kbd> — bleed

Bleed means artwork extends beyond the trim line so color reaches the physical edge after cutting.

<Diagram
  label="Bleed Map"
  src="/diagrams/worksheet/f-bleed.svg"
  alt="Bleed, trim line, and safe area with legend"
  caption="Draw full-page backgrounds on the bleed layer so edge-to-edge print works."
/>

```js
function setup() {
  book = createBook(4, 6, 8);
  book.setBleed(0.125); // 1/8 in (or 3 mm)
}

function draw() {
  book.bleed.background(220, 60, 80); // bleed layer

  clear(); // keep trim area transparent so bleed is visible
  fill(255);
  textAlign(LEFT, TOP);
  textSize(20);
  text(book.pageNumber, 20, 24);

  book.addPage();
}
```

## <kbd>G</kbd> — typography

Typography can be the main structure of a book. Instead of drawing many objects, you can let one word, one letter, or one typographic rule change page by page.

<div class="step" data-step="1">
Start with one letter per page.

```js
const WORD = "WORKSHOP";

function setup() {
  book = createBook(4, 6, WORD.length);
}

function draw() {
  background(book.page % 2 === 0 ? 0 : 255);
  fill(book.page % 2 === 0 ? 255 : 0);

  textAlign(CENTER, CENTER);
  textFont("Georgia");
  textStyle(BOLD);
  textSize(width * 0.52);
  text(WORD[book.page], width / 2, height / 2);

  book.addPage();
}
```

</div>

<div class="step" data-step="2">
Add rhythm with tracking, baseline shift, or size changes:

```js
let s = map(book.progress, 0, 1, width * 0.4, width * 0.72);
textSize(s);
book.letterSpacing(book.page % 2 === 0 ? -1 : 3);
```

</div>

<div class="try">

- Try a two-font system: one font for cover, one for interior pages.
- Try a strict rule: odd pages always dark, even pages always light.
- Try spelling a phrase across a spread by coordinating left and right pages.

</div>

Try: [typography](/examples/04-typography) and [type rhythm](/examples/14-type-rhythm).

## <kbd>H</kbd> — generative patterns

Generative books are systems: the page is the time axis, and your rule evolves across that axis.

<div class="step" data-step="1">
Pick one pattern rule and keep it simple:

```js
function draw() {
  background(255);
  randomSeed(book.pageNumber * 99); // stable pattern per page

  for (let i = 0; i < 120; i++) {
    stroke(0, 30);
    line(random(width), random(height), random(width), random(height));
  }

  book.addPage();
}
```

</div>

<div class="step" data-step="2">
Then change exactly one parameter across pages:

```js
let density = floor(map(book.progress, 0, 1, 40, 240));
let strokeA = map(book.progress, 0, 1, 12, 48);
```

</div>

<div class="try">

- Keep the same geometry, but shift only density.
- Keep density fixed, but shift only color.
- Compare two runs: deterministic (`randomSeed`) vs every run different.

</div>

Try: [every run different](/examples/07-every-run-different), [slow burn](/examples/06-slow-burn), and [pattern layers](/examples/15-pattern-layers).

## <kbd>I</kbd> — make your book

Now combine what you built into one finished mini-book.

Pick one direction and finish it:

- **Type system book:** one word/letter per page, strong typographic rhythm.
- **Pattern book:** one generative rule, gradual parameter shift.
- **Story book:** explicit cover, interior narrative, back cover.
- **Print-minded book:** include bleed and test a printed prototype.

<div class="step" data-step="1">
Define your constraint in one sentence:

- “One letter per page with a strict black/white rhythm.”
- “One pattern system that increases in density.”
- “Story arc: cover, interior, back cover.”

</div>

<div class="step" data-step="2">
Build a first complete pass quickly, then polish one layer:

- first pass: structure + export works
- polish pass: typography, spacing, or color rhythm

</div>

Minimum done definition:

- Has cover + back cover.
- Uses page progression (`pageNumber`, `page`, or `progress`).
- Has at least one intentional visual rule.
- Exports successfully as PDF.

<div class="try">

- Export PDF once early; don’t wait until the end.
- If time is short, cut scope to 8–12 pages and finish a complete book.
- Print one draft if possible and adjust contrast/size for readability.

</div>

Try: [cover & back story](/examples/12-cover-story), [workshop remix](/examples/13-workshop-remix), and [data-driven book](/examples/09-data-driven-book).

## <kbd>K</kbd> — cheat sheet, troubleshooting, links

### quick cheat sheet

|                                |                                                        |
| ------------------------------ | ------------------------------------------------------ |
| **in setup() — run once**      |                                                        |
| `createBook(4, 6, 8)`          | 4 in wide, 6 in tall, 8 pages — canvas auto-created    |
| `createBook("A5", 8)`          | standard paper size (A3 A4 A5 A6 letter legal tabloid) |
| `book.setDPI(300)`             | high-resolution export — for printing                  |
| `book.setBleed(0.125)`         | 1/8 inch bleed — or `(3, "mm")` for European standard  |
| `book.setSpread(true)`         | pair interior pages as spreads                         |
| **in draw() — captured pages** |                                                        |
| `book.pageNumber`              | current page — starts at 1                             |
| `book.page`                    | same, starts at 0 — use as an array index              |
| `book.totalPages`              | total pages                                            |
| `book.progress`                | 0 on page 1 → 1 on last page                           |
| `book.isFirstPage()`           | true only on the first page                            |
| `book.isLastPage()`            | true only on the last page                             |
| `book.bleed.background(c)`     | full-bleed background — extends past trim              |
| `book.addPage()`               | save this page — always put this **last**              |

### troubleshooting

- **Only one page exported:** `book.addPage()` is in `setup()`; move it to `draw()`.
- **Blank/empty PDF pages:** make sure you draw before `book.addPage()`.
- **`isLeftPage()` warning:** call `book.setSpread(true)` in `setup()` first.
- **Print blocked in Arc (`ERR_BLOCKED_BY_CLIENT`):** use **Download → PDF** and print the file, or disable Shields for the p5 editor tab.
